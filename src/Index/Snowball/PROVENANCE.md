# Vendored Snowball stemmers — provenance

The PHP files in this directory are **generated code**, produced by the
Snowball compiler's `-php` backend and vendored here so the indexer's
build-time stems match Pagefind's query-time stems byte-for-byte. Do not edit
them by hand — regenerate with `scripts/generate-stemmers.sh` (which applies
the transforms below deterministically) and re-baseline the manifest.

## Why a mainline commit, not a release tag

`pagefind_stem` 1.0.0 vendors Rust sources generated by the Snowball compiler
from an unpinned snowball checkout (see the crate's own `build.js`). The
commit above was recovered by byte-comparing the compiler's `-rust` output
against the crate's vendored sources: at `019c1bd` **all 34 generated
algorithm files and the 3 Rust runtime files are byte-identical** to
`pagefind_stem` 1.0.0, and the algorithms directory contains exactly the
crate's 34-algorithm feature set (no czech/persian/sesotho, which landed
later). Any commit producing identical output would be an equally valid pin;
this one is verified.

Neither adjacent release tag is parity-correct:

- **v3.0.0** predates the `english.sbl` fixes the crate has — it diverges on
  18 English words (`internal`, `interval`, `interfered`, `skis`, …).
- **v3.1.1** postdates apostrophe/elision changes the crate does NOT have
  (Danish/Norwegian/Finnish apostrophe handling, Italian elisions, German
  apostrophe) — generated at v3.1.1, da/fi/it/no diverge from Pagefind on
  2,103 corpus words, and de diverges on apostrophe words the corpus happens
  not to contain.

Parity with what Pagefind's WASM actually runs beats "newer algorithm": a
stem Pagefind cannot reproduce at query time is a word nobody can find.

## The version-header trap

Each generated file carries a `Generated from <x>.sbl by Snowball <version>`
header. That string is the **compiler's version at the pinned commit**
("3.0.0" here, since the commit predates the 3.1.0 bump) — it identifies
neither a release tag nor the algorithm state. Never trust the header: verify
the commit hash above, and let the corpus parity tests judge the output (0
divergences over 589,069 words across all 14 languages at this baseline).

## Transforms applied by the generation script

Mechanical, idempotent, applied identically to every file:

1. `namespace Tag1\Scolta\Index\Snowball;` inserted after the opening
   `<?php` (the compiler emits global classes).
2. Typed class constants (`private const array A_0`, PHP 8.3+) stripped to
   untyped (`private const A_0`) — this package's floor is PHP 8.1. Values are
   unchanged; the corpus byte-parity tests validate the transformed code.
3. The compiler's hardcoded `Snowball` class-name prefix dropped
   (`class SnowballEnglishStemmer` → `class EnglishStemmer`) for PSR-4-clean
   names. The runtime base class keeps its upstream name, `SnowballStemmer`
   (from `php/base-stemmer.php` at the same commit).

The script also greps the output for post-8.1 syntax and fails if any remains.
